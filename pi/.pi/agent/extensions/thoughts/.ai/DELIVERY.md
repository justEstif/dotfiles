## Architecture

### Three Entry Types
1. **ThoughtAnchor** (`kind: "start"`)
   - Marks thread start with snapshot + display name
   - Generated by `/thought:start`

2. **ThoughtLabel** (`kind: "label"`)
   - Checkpoint within a thread
   - Generated by `/thought:label`

3. **ThoughtSummary** (`kind: "summary"`)
   - Pre-generated on `turn_end`
   - Returned at `session_before_tree` navigation
   - Placeholder text ready for LLM calls

4. **ThoughtEnd** (`kind: "end"`)
   - Resolution marker
   - Generated by `/thought:end`

### Key Design Decisions
- **Label convention**: `label: "thought:<slug>"` (no separate index file)
- **Custom entries**: Snapshots stored outside LLM context → survives compaction
- **Cross-pwd discovery**: `SessionManager.listAll()` scans all sessions globally
- **Tree integration**: Branches are pi's native `/tree` navigation; summaries returned at fork points
- **Opt-in only**: No auto-detect; explicit `/thought:start` required

## Usage Flow

```
1. /thought:start "Lead with 1G or 500?"
   → Captures snapshot, creates anchor

2. (work for a few turns)

3. /thought:label "rejected 500 strategy"
   → Checkpoint for this decision

4. /tree → navigate to alternative branch
   → Abandoned branch gets pre-generated summary

5. /thoughts
   → Lists all threads (including from other cwds)

6. /thought:resume "lead-with-1g-or-500"
   → Switches to that session

7. /thought:end "Decided to lead with 1G"
   → Marks thread resolved

8. /thought:view "lead-with-1g-or-500"
   → Shows full thread with summary + resolution
```

## How Summaries Work

### Generation (turn_end hook)
```typescript
pi.on("turn_end", async (event, ctx) => {
  // After each turn, check if an active thought thread exists
  // If so, schedule background summary generation (non-blocking)
  // Max 1 summary in-flight at a time
});
```

### Retrieval (session_before_tree hook)
```typescript
pi.on("session_before_tree", async (event, ctx) => {
  // When leaving a branch, return pre-generated summary
  // No waiting on model; summary already in session storage
});
```

## Storage Format

Everything persists in `~/.pi/agent/sessions/<cwd>/<uuid>.jsonl`:

```jsonl
{"type":"label","id":"abc","label":"thought:lead-with-1g-or-500"}
{"type":"custom","id":"def","customType":"thoughts","data":{"kind":"start",...}}
{"type":"custom","id":"ghi","customType":"thoughts","data":{"kind":"summary",...}}
{"type":"custom","id":"jkl","customType":"thoughts","data":{"kind":"end",...}}
```

No separate index files. No sync issues. Standard JSONL format.

## Development Notes

### Extension Structure
```

├── package.json           # metadata + pi extension marker
├── types.ts               # discriminated union for 4 entry types
├── index.ts               # all commands + hooks
```

### Key Helpers
- `captureSnapshot(ctx, leafId)` — Extracts 3-message context
- `findThoughtAncestor(ctx, fromId)` — Walks branch for active thread
- `generateSummaryInBackground(pi, ctx, rootLabel, leafId)` — Ready for LLM calls
- `slugify(name)` — Converts display names to slug form
- `validateThoughtName(name)` — Rejects reserved stems + topics
