## Routing

1. **Product / startup / app idea to validate?**
   → **MANDATORY READ** `references/build-or-not.md`. Three gates: north star, core tech, defining constraint.

2. **Specific plan or design to resolve end-to-end?**
   → **MANDATORY READ** `references/grill-me.md`. Dependency-ordered interview until every branch is decided.

3. **Organizational complexity, feedback loops, multi-system dynamics?**
   → **MANDATORY READ** `references/systems-thinking.md`. Rumsfeld Matrix + Causal Loop Diagrams + debt cascades + fence investigation.

4. **Pushback on a claim, or want general adversarial thinking?**
   → **MANDATORY READ** `references/sycophancy.md`. Defaults to constructive disagreement. Includes YAGNI-at-scale pushback against hedging.

5. **Received a request that might not be the real need?**
   → **MANDATORY READ** `references/root-ask.md`. Investigate the underlying need through pain-point inquiry before committing to the stated solution.

6. **Need to get organizational buy-in before a meeting?**
   → **MANDATORY READ** `references/stakeholder-prealignment.md`. Nemawashi: systematic one-on-one pre-alignment so the meeting becomes ratification, not debate.

7. **Ambiguous?**
   → Default `sycophancy`. Switch if the exchange reveals a clearer fit.

## Sequencing

Multiple modes apply? Run structuring → specific → granular:

**systems-thinking** → **build-or-not** → **root-ask** → **grill-me** → **stakeholder-prealignment** → **sycophancy**

One mode per turn. Complete its line of questioning before switching.
