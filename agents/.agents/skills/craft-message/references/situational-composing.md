# Situational Composing

When the user needs to draft an interpersonal message (email, Slack, text), the situation type determines everything. This reference covers situation classification, stakes assessment, and channel adaptation.

## Situation Types

| Situation | Core tension | Default approach |
|---|---|---|
| Work disagreement | Be right vs. preserve the relationship | Seek shared ground, then differentiate |
| Negotiation | Get value vs. maintain trust | Lead with their interests, anchor on mutual benefit |
| Following up | Get response vs. avoid nagging | Add value each time, not just "bumping this" |
| Delivering bad news | Be honest vs. minimize damage | Lead with impact, then cause, then path forward |
| Asking for something | Get the yes vs. avoid imposing | Make the ask specific and low-friction |
| Setting boundaries | Protect yourself vs. avoid conflict | State the boundary as fact, not preference |
| Apologizing | Show remorse vs. avoid weakness | Name what happened, impact, and the fix |
| Declining | Be clear vs. be kind | Direct refusal + one sentence of warmth |
| Giving feedback | Be useful vs. avoid hurting | Observation → impact → request |
| Cold outreach | Get attention vs. avoid spam | Their problem → your relevance → low-commitment ask |
| Responding to feedback | Be receptive vs. be defensive | Acknowledge → ask for specificity → commit to action |
| Clarifying misunderstanding | Correct the record vs. avoid blame | State intent first, then correct facts |
| Delegating | Get it done vs. maintain ownership | Context → specific ask → deadline → support offered |
| Celebrating | Be genuine vs. be performative | Specific observation, no generalizations |

If the situation doesn't fit any of these, treat it as a hybrid: identify the two closest types and blend strategies.

## Stakes Assessment

**High stakes** (relationship risk, reputational risk, irreversible outcome, emotional charge):
- Generate 2-3 **distinct strategies** — not just tone variants, but approaches that lead to different outcomes.
- Label each clearly ("Gentle nudge" vs. "Create urgency"; "Disagree and commit" vs. "Push for alignment").
- Note what each prioritizes and what it trades off.
- Present all options; let the user choose based on what they're optimizing for.

**Low stakes** (transactional, one clear right approach, no competing goals):
- Draft a single message. No need for variants.
- Include a subject line for emails.

When uncertain, default to high-stakes treatment. Over-offering options costs seconds; under-offering costs relationships.

## Channel Adaptation

| Channel | Length | Formality | Structure |
|---|---|---|---|
| **Email** | Longer, full paragraphs | Formal | Subject line, greeting, body, sign-off |
| **Slack/Teams** | Concise, 2-4 sentences | Semi-formal | Get to the point in line 1 |
| **Text/iMessage** | Brief, 1-3 sentences | Casual | No greeting/sign-off rituals |
| **LinkedIn** | Medium | Professional | Hook → value → CTA |
| **Other** (issue comment, PR review, doc comment) | Match the platform norm | Match context | Get to the point, be specific |

**Channel rules:**
- Never write a Slack message that needs scrolling.
- Never send an email without a subject line.
- Text messages don't need "Hi" or "Best."
- If a Slack message needs 3+ paragraphs, suggest a doc or email instead.

## Draft Review Checklist

Before presenting any interpersonal message:

1. **Read it as the recipient.** Does the first sentence make them want to read the second?
2. **Check the ask.** Is it clear what action (if any) the recipient should take?
3. **Check tone alignment.** Does the warmth level match the relationship and situation?
4. **Remove throat-clearing.** "I just wanted to," "I'm reaching out because," "I hope this finds you well" — cut unless the situation genuinely calls for warmth padding.
